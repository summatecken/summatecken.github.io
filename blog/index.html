What do you think of pre-compiling your CSS?

Compare to regular CSS

I definately can see a huge difference in maintainability and reduced time to setup styling. I know that when I first learned about CSS I thought it was a very tedious process to write for example ul li a {} to change link elements in a list, and then if I wanted to do a hover effect I'd write ul li a:hover {} which made the style sheet pretty cluttered. But with pre-compiled CSS it is easier to read and understand.

Which techniques did you use?

In this project I mostly used just nesting and variables, since I didn't find it too usable to use modules for example as this is a very basic blog site. I did try some other functionality such as @extend and operators and can absolutely see benefit in using these.

Pros and cons?

Pros are speed of development, maintenance and reduced clutter. Cons are that you need additional software (which can be installed in for example Node.js) to make use of it, while regular CSS needs no additional setup.

What do you think of static site generators?

What type of projects are they suitable for?

The way I understand it is that it works good for quick jobs, for example a client who wants a website that displays opening hours and contact information for a restaurant, and then it would be very easy to update the information by editing the markdown files if there are any changes in the schedule. I think that quick setup and flexible dynamic content are key points here.

What is robots.txt and how have you configured it for your site?

It's a file that tells web crawlers / indexers about what content is allowed to be fetched. From what I understand, having a robots.txt does not work for this assignment since it requires to be placed in the top domain's root, which is not the case in this assignment as we use github pages. Instead I've added meta tags for name=robots, which has the rules "noindex" and "nofollow". However, having such rules does not apply to all bots, it only applies to those that conforms to those rules.

What is humans.txt and how have you configured it for your site?

It's a text file that lists all people involved with the website, and optionally software and other information can be added too. It is not required and not all sites use it. I have added a link tag to it, but it is not accessible as with the robots.txt, since it has to be placed in the top domain's root.

What is Open Graph and how do you make use of it?

It is is a way of telling how a website's thumbnail should be displayed when shared. It can have a custom title, description and image for example. I've added the properties: title, type, url and image.
